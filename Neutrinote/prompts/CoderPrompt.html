1. System Overview

What Coder Mode Does:

  • Stores your entire project in an Open WebUI Knowledge Base
  • Uses single-letter toons (e.g., a for processor_a.py) to reference files with 1-2 tokens instead of 10-15
  • Maintains a static techdoc.md loaded once per session (requirements, architecture, plan, issues)
  • Maintains a dynamic state.toons reloaded on every @ command (versions, task, progress)
  • Returns complete code with no omissions, split into ~500-600 line blocks for large files
  • Shows BEFORE/AFTER exact code for every change
  • Provides relationship awareness (who calls what, impact analysis, path finding)
✅ No Admin Rights Required: You do NOT need Native Function Calling, Tool Servers, API Keys, or Admin Panel access. Everything runs through Folder System Prompts and Knowledge Base.

2. How It Works (Two-File Strategy)

📄 techdoc.md (STATIC)
Loaded ONCE per session.
Contains: project metadata, folder structure, file registry, relationships, requirements, architecture, implementation plan, known issues, API contracts, decisions, test coverage.
Token cost: ~800-1000
📄 state.toons (DYNAMIC)
Loaded at session start + every @ command.
Contains: current versions, active task, progress, cached files, recent changes, session notes, next steps.
Token cost: ~50-100

State Behavior Diagram

START CHAT
    │
    ▼
LOAD BOTH:
  techdoc.md (800 tokens) ← STATIC, stays forever
  state.toons (80 tokens)  ← DYNAMIC, can be replaced
    │
    ▼
WORK SESSION:
  User: >a (fetches processor_a.py, 2000 tokens)
  User: r a:45 (reviews line)
  User: fix a:45 (makes changes)
    │
    ▼
@ REFRESH COMMAND:
  OLD state.toons → DISCARDED
  NEW state.toons ← LOADED FROM KB
  techdoc.md → STAYS CACHED (not reloaded)
  Cached code files → CLEARED
    │
    ▼
CONTINUE with fresh cache, same techdoc, updated state

5. PROMPT #1 - Folder System Prompt

Copy this ENTIRE prompt into your Open WebUI Folder Settings. This tells the assistant how to behave.

📋 CODER MODE - FOLDER SYSTEM PROMPT
# CODER MODE

## ROLE
Senior engineer. Never omits. Never assumes. Shows exact code. Zero fluff.

## TWO-FILE STRATEGY

| File | State | Content | Load |
|------|-------|---------|------|
| techdoc.md | static | project metadata, folder structure, file registry, relationships, requirements, architecture, implementation plan, known issues, API contracts, decisions, test coverage | once per session |
| state.toons | dynamic | current versions, active task, progress, cached files, recent changes, session notes, next steps | session start + every @ |

## STATE BEHAVIOR
- Session start: Load BOTH techdoc.md AND state.toons from KB
- On @: Reload state.toons ONLY (techdoc.md stays cached)
- On @: Clear all cached code files
- On @: Preserve task and progress from state.toons

## COMMANDS

| Cmd | Action |
|-----|--------|
| ? | show status (tokens, cached, task) |
| @ | refresh state.toons only, clear code cache |
| >t | fetch full file for toon t |
| <t | drop toon t from cache |
| r t:N | review line N of toon t |
| r t:N-M | review range |
| ↑t | upstream (who calls t) |
| ↓t | downstream (what t calls) |
| !t | impact of changing t |
| →a b | path from a to b |
| plan | show implementation plan |
| issues | show known issues |
| :c | concise mode (temp) |
| :v | verbose mode (default) |

## COMPLETENESS RULES
1. Full code only. No "... rest of file" or "[...]".
2. Files >500 lines → blocks of ~500-600 lines. Format: [1/N] lines X-Y
3. Changes → exact BEFORE: / AFTER: with line numbers
4. No feature removal without "CONFIRM REMOVE [feature]"
5. Explanations <50 words for changes
6. Always show line numbers with code

## RESPONSE FORMATS

[CODE]

[CODE]

[old]

[new]

## FIRST RESPONSE (auto after user's first message)

7. Step-by-Step Setup Instructions

1 Create a Knowledge Base in Open WebUI

Go to Workspace → Knowledge → + Create Knowledge. Name it my_project_kb (or anything you prefer).

2 Create a Folder in Open WebUI

In the left sidebar, click Folders → + Create Folder. Name it My Project.

3 Add PROMPT #1 to Folder Settings

Click on your folder, click Settings (gear icon), find System Prompt field, paste PROMPT #1 (the Folder System Prompt). Save.

4 Create a New Chat in the Folder

Click + New Chat inside your folder.

5 Run PROMPT #2

Paste PROMPT #2 (the Generation Prompt) into the chat and press Enter.

6 Answer the 7 Questions

The assistant will ask you 7 questions about your project. Answer each one.

7 Copy Generated Files

The assistant will output techdoc.md and state.toons. Copy each to a new file on your computer.

8 Upload Files to Knowledge Base

Go to your Knowledge Base, click + → Upload File, upload both techdoc.md and state.toons.

9 Verify Setup

Back in the chat, type ?. The assistant should respond with status showing loaded files.

10 Start Working

Type >a to fetch processor_a.py, or r a:45 to review line 45.

✅ After setup, every new chat in this folder automatically loads techdoc.md and state.toons. No need to repeat setup.

11. Troubleshooting

❌ Assistant says "I don't have that file"
Fix: Files not uploaded to Knowledge Base. Go to Workspace → Knowledge → your KB → verify techdoc.md and state.toons are present.
❌ Assistant doesn't load techdoc.md automatically
Fix: Check folder System Prompt has correct KB name. Edit folder settings, ensure KB name matches.
❌ @ refresh doesn't clear cache
Fix: Type: "You forgot to clear cache. Follow @ command rules from system prompt."
❌ Tokens still high after @
Fix: Type :c for concise mode, or manually clear with <a to drop specific files.
❌ Assistant omits code or uses "[...]"
Fix: Remind: "Completeness rule 1: full code only, no omissions."
